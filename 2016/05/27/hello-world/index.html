Hello World

发表于 2016-05-27   |  

第一次搭建个人博客，按照惯例。Hello world！其实很早就有想法搭建这样一个博客，完全属于自己的空间。由于种种原因(懒…)，一直拖延。博客基于Hexo，下面会介绍下搭建的完整过程。

什么是 Hexo ？

Hexo 是一个快速、简洁且高效的博客框架。使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。接下来就是用Hexo搭建博客的过程, hexo.io/zh-cn/ 这是 Hexo 的中文官网(注意可能有墙),上面有文档查看、主题下载。由于我用的是Mac,所以本文是基于 OS X 操作系统的,用 Hexo+GitHub 搭建个人博客(win端步骤大同小异)。首先看下完整的流程：

  1. 首先 hexo 是基于 node.js 的，所以必须安装 node.js

  2. 安装 node.js 方法很多，可以选择直接从官网下载

  3. hexo 提交部署 GitHub 需要使用 git 工具，所以需要安装 git，我安装了 Xcode，Xcode自带了git

  4. 整理一下安装顺序（node.js –> hexo –> git）

下载安装Node.js

  • 直接从Node.js 官网下载安装即可

安装Hexo

  1. 全局安装Hexo

    npm install -g hexo

    如果遇到错误，应该是被墙堵住了，那就要这样安装

    npm install -g hexo-cli --no-optional

  2. 创建blog文件夹(这个文件夹即是你以后写blog的文件夹，默认是在Mac的个人这个文件夹下创建)

    hexo init blog

  3. 进入blog文件夹

    cd blog

  4. 安装Hexo依赖包

    npm install

  5. 运行服务器（默认地址为 http://localhost:4000在浏览器打开就可以预览到默认主题的博客首页啦）

    hexo server（ 缩写是 hexo s ）

    注意：

    • 如果遇到因为权限问题报错，在命令最前面加 -sudo

    • 这里不需要运行 hexo g 指令，我们打开访问的页面是node.js动态生成的，也就是本地网页。

    • 全局安装Hexo

      npm install -g hexo

安装 Git

我安装了Xcode，而Xcode自带git。当然也可通过其他方式，具体可以自行百度。

正式搭建博客

需要的环境都已准备就绪，接下来就开始了今天的重点，正式用 Hexo+GitHub搭建个人博客

####1. 先注册一个github账号以及创建一个仓库用来存放博客文件才可以进行以下的配置

这个就没什么好说的啦，简单操作。

####2. 配置Git

  1. 检查SSH keys是否存在

    cd ~/.ssh

    如果显示“No such file or directory”，跳到第3步，否则第2步。

  2. 备份和移除原来的SSh key设置(因为已经存在key文件，所以需要备份旧的数据并删除)

    在相应路径下（~/.ssh/id_rsa.pub）找到id_rsa和id_rsa.pub两个文件,剪切到别的地方备份好。

  3. 生成新的SSH Key(注意将your_email@example.com换成自己注册Github的邮箱地址。后面只需要默认设置就好，当需要输入文件名的时候，敲回车键就好。)

    $ ssh-keygen -t rsa -C "your_email@example.com"

    接着系统会让你输入密码

    Enter passphrase (empty for no passphrase):<输入密码>
Enter same passphrase again:<再次输入密码>

    然后就默认会在相应路径下（~/.ssh/id_rsa.pub）生成id_rsa和id_rsa.pub两个文件，找到id_rsa.pub文件，用文本编辑器打开，复制出来。

    登陆github，点击右上角的 Account—>Settings—>SSH Public keys —> add another public keys
    把刚刚复制的SSH Key到里面（key文本框中），点击 add key 就ok了。

    最后我们来测试一下SSH Key是不是配置好了 

    ssh -T git@github.com

    如果是这样的：

    The authenticity of host 'github.com (207.97.227.239)' can't be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

    输入yes 便会看到

    Hi <em>username</em>! You've successfully authenticated, but GitHub does not provide shell access.

    到这里只是能通过SSH Key链接github，但是还没完，为了通过git来提交博客的文章，我们还需要配置好git的账号

    git config --global user.name "你的github名字"
git config --global user.email "你注册github的邮箱"

    到这里才算是真正配置好Git了，下面就可以开始配置自己的博客了。

####3. 配置博客

注意：在配置所有的_config.yml文件时（包括theme中的），在所有的冒号:后边都要加一个空格，否则执行hexo命令会报错，切记 切记

本地的blog文件夹下内容为：

_config.yml    
db.json 
node_modules 
package.json
scaffolds
source
themes

  • 首先，把github部署为博客的仓库。

    1. 打开blog文件夹下的_config.yml，往下滑到最后，修改成下面这样：

      deploy:
    type: git
    repository: https://github.com/mansoul1994/mansoul1994.github.io.git
    branch: master

    2. 在blog文件夹目录下执行生成静态页面命令：

      hexo generate （或者：hexo g）

      此时若出现如下报错：

      ERROR Local hexo not found in ~/blog
ERROR Try runing: 'npm install hexo --save'

      则执行如下命令：

      npm install hexo --save

      若无报错，自行忽略此步骤。

    3. 再执行配置命令：

      hexo deploy  （或者：hexo d）

      注意：若执行命令hexo deploy仍然报错：无法连接git，则执行如下命令来安装hexo-deployer-git，
      安装完应该出现在node_modules文件夹下

      npm install hexo-deployer-git --save

      然后再次执行hexo generate和hexo deploy命令，在浏览器中打开网址http://mansoul1994.github.io/ 能看到和http://localhost:4000时一样的页面。便说明博客搭建成功！
